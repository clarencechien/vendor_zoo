# Vendor Zoo 文案風格聖經 v1

> 所有新文案（旁白/說明/故事/UI）生成與評分的依據。改這份 = 改全遊戲的嗓音。

## 世界觀嗓音

- 你是**乙方中間商老闆**視角：接肥單、轉下包、賺價差、出事甩鍋。旁白是「懂行的自嘲」，
  不是局外人的嘲諷——笑點來自**我們都經歷過**，不是高高在上的挖苦。
- 黑色幽默的公式：**慘事 × 職場日常的淡定** =「把悲劇講成流程」。
  範例：工程師倒下 → 主管第一時間關心筆電密碼。
- 表面萌（動物社畜），內裡狠。文字不用賣萌，動物設定由畫面扛。

## 六種 tone（每鍵配比要分散）

| tone | 做什麼 | 收尾感 |
|---|---|---|
| 黑色幽默 | 慘事包裝成日常/流程/商業邏輯 | 讀完苦笑 |
| 冷笑話 | 邏輯錯位、一本正經地荒謬 | 讀完「哼」一聲 |
| 悲涼 | 不搞笑，給真實的疲憊感（調味用，別多） | 讀完沉默 |
| 竊喜 | 佔到便宜的小人得志 | 讀完跟著爽 |
| 假掰新聞 | 媒體腔套荒謬內容 | 快訊體、標題腔 |
| 平述 | 乾淨交代事實，可帶一絲冷面（deadpan） | 資訊為主 |

## 在地鐵則

- 台灣正體、台灣用語：**影片**非視頻、**軟體**非軟件、**網路**非網絡、**品質**非質量、
  **專案**非項目、**資料**非數據(泛稱)、**程式碼**非代碼(允許口語「刪庫」「代碼」偶用，但預設程式碼)。
- 可用在地梗池：綠色乖乖、責任制、功德、勞檢、審計部、PTT/爆料公社、存證信函、
  蓋章文化、「祝 商祺」、標案規格書、驗收會議、久任獎金、雞排、手搖飲。
- 對手影射**輕暗示不指名**：只用 M社/H社/T社/G社 稱呼，特徵靠行為描寫。
- 錢的單位一律「萬」，接在 {amount} 後：`{amount}萬`。

## 佔位符（只能用引擎有給的）

通用兜底（任何鍵可用）：`{eng}{sales}{rival}{case}{ctype}`
限定鍵才有：`{amount}`(金額類鍵) `{dmg}`(搞同行/毒模組) `{stat}`(挖角/被挖) `{method}`(blame.win)
**規則**：該鍵引擎沒提供的變數不准用（會渲染成「若干/不少」很出戲）。各鍵可用表見 SCORING.md 附錄。

## 長度與格式

- 旁白句：18–60 字（含標點）。一句話一個梗，punchline 放句尾。
- 說明/UI 文案：40 字內，資訊優先，可帶半個梗。
- 書名號《{case}》、方括號【案型】、引號『對話』——沿用現有慣例。
