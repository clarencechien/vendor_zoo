# 文案評分方式 v1（先驗收這份，再開始大量評分）

## 硬性關卡（lint，不過就出局，不進評分）

1. 佔位符合法：只用該鍵引擎提供的變數（附錄A）；渲染測試零殘留 `{xxx}`。
2. 禁用詞零容忍：視頻/軟件/網絡/質量(當品質用)/項目(當專案用)/數據庫/信息/屏幕/優化(濫用時)…
3. 長度：旁白 18–60 字；說明 ≤40 字。
4. 撞梗檢查：與同鍵現有句的 4-gram 重疊 <50%，且核心梗不重複（人工判）。

## 五維評分（各 0–5）

| 維度 | 代號 | 問的問題 |
|---|---|---|
| 黑色幽默 | 黑 | 有沒有做到該 tone 該做的事？（平述句評 deadpan 乾淨度、悲涼句評餘韻） |
| 在地感 | 地 | 台灣 IT/外包/公部門的人看了會不會說「太真實」？ |
| 新鮮度 | 鮮 | 是不是句庫裡沒有的新梗？老梗（如單純「加班慘」）扣分 |
| 清晰度 | 清 | 一秒看懂發生了什麼事、跟遊戲狀態變化對得上 |
| 節奏 | 節 | 斷句順口、punchline 在句尾、沒有贅字 |

## 加權（兩種 profile）

- **旁白句**（narration 46 鍵）：黑30% + 地25% + 鮮20% + 清15% + 節10%
- **說明文案**（UI/系統訊息/選項副標）：清40% + 地20% + 節20% + 黑15% + 鮮5%

## 入庫門檻

- 加權 **≥ 3.6** 且 清 **≥ 3**（旁白）；說明文案加權 ≥ 3.8。
- 每鍵 tone 分布：同鍵內同 tone 不超過一半。
- weight 欄給法：加權 ≥4.3 → weight 4；3.6–4.29 → weight 3；悲涼/平述一律 ≤3。
