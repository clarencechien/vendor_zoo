# 外包生存模擬器 —— Web 版規格書 (SPEC v1，對應引擎 v0.3)

> 用途：這份是 **JS 重寫版的施工藍圖 + 驗收清單**。
> 開發順序：**①本 spec✅ →②wireframe(進行中) →③RWD/PWA mockup(prototype UI，靜態假資料) →④才動手寫 JS 引擎接上 UI**。
> 技術定案：**純 HTML/CSS/JS 單一檔 + RWD + PWA(輕量)**；視覺定案：**像素立繪+復古風**(mockup 先 emoji 佔位)。
> Python 版之後只作為「平衡實驗室」(grid/score_balance)；出貨的是 JS 版。
> **平衡數值以 Python 版 CONFIG/DIFF 為唯一真相來源**，JS 版頂端放對應區塊，改動時同步。

## 0. 一句話定義

你是政府「旗艦計畫」五家龍頭之一的**外包中間商老闆**。這是**大逃殺**：撐過 15 季，或搞死其他四家獨活。核心動詞是**搞同行**（檢舉／放假消息／挖角／塞毒模組）。你搞的每一手都記帳——技術債會爆、團隊會崩、稽核會來、被搞的對手會記仇反撲。

## 5. 回合結構

每季流程：
1. **季初**：業界快訊(旁白) → 擲事件(60%觸發，22%再一發，冷卻3季) → 顯示儀表板。
2. **行動階段**：花 5 點行動點，玩家自由操作。
3. **季末結算**：風險成長 → 維護收支 → 客戶倒閉/提告 → 案子引爆處理 → 對手自然消長/倒閉/接盤 → **對手仇恨反撲** → 稽核(到期時) → 判定死活。

## 6. 核心公式

### 擲骰 roll(bonus, dc)
`d = 1..20；成功 = (d==20) 或 (d!=1 且 d+bonus>=dc)`。d20=大成功、d1=大失敗。

### 搶標
`bonus = max(業務能力) + bid_buzzword`
`dc = bid_dc_base + load×bid_dc_per_load + max(0,存活對手-1)×bid_dc_per_rival + ⌊案數/2⌋`
成功：期初款 `randint(110,220) × D(bid_income)`(大成功×1.5)，案子下包成立。

### 話術-榨錢
冷卻檢查(same case < upsell_cooldown 拒收)。`gain = randint(upsell_base) × 案型upsell_mult`(大成功×1.4)；`sat -= 案型upsell_sat`。

### 救火
`roll(max(工程師能力), rescue_dc)` 成功→`risk -= randint(rescue_risk_cut)`、`sin -= rescue_sin_cut`、`morale -= 2`。

### 甩鍋（真兩難，爆炸時 or 手牌）
DC = `11 + 案型blame_dc`(+信譽/滿意度懲罰)：
- **成功**：不賠錢，但 `sin += blame_sin_base × audit_w × 1.6`(養稽核)、`risk -= blame_risk_cut`、`sat -= 12`。
- **失敗**：立刻賠 `randint(120,220) × D(penalty) × explode_mult`、`reputation -= 8`、`sat -= 10`、`sin += blame_sin_base × audit_w × 0.4`。
- **(爆炸選單另有)硬扛**：賠 `randint(80,180) × explode_mult × D(penalty)`，`risk=40`，`sin -= 4`(不養稽核)。
- **(爆炸選單)推替死鬼**：燒一名工程師，`risk=30`、`morale -= 15`、罪孽不增。

### 季末維護收支
每案：`base=45；risk_factor = (risk<40? 1.0 : max(-0.15, 1-(risk-40)/55))`；
`income += 45 × income_mult × risk_factor × (lockin?1.8:1)`。→ **壓在低風險才全額收，冒煙會失血**。
`cash += 總income − 薪資租金 − (存活對手數 × rival_pressure)`。

### 風險成長
每季每案 `risk += randint(risk_rise_base)`；達 100 引爆。

### 稽核（延遲帳單）
每 audit_interval 季一次。`d20 vs 罪孽壓力(sin 超過 audit_sin_soft 的部分放大)`。過關洗少量sin；未過 `罰款 = (randint(audit_penalty)+(sin-soft)×3) × D(audit)`，信譽-15，重罪→勒令解散(稽核死)。

## 7. 事件系統

### 事件袋 Bag（抽牌不放回 + 冷卻）
`items` 洗牌成袋，逐一 pop；袋空重洗；每個事件 key 有 cooldown 季不重抽。**11 個事件**：工程師中風、刪庫跑路、業務捲款、勒索軟體、甲方倒閉、雲端當機、套件刪庫、長官視察(crash/ok)、王牌被挖角、媒體爆料、天上掉錢。

### 旁白系統 Narrator（把引擎事件演成台灣味黑色幽默）
- 資料：`context_key, tone, weight, template`，共 **270 條 / 46 情境鍵 / 每鍵3-6句**。
- 抽句：**加權挑選 + 避開最近用過的**(recent 窗 = min(句數-1,4))——不會短期重複(這是 v0.3 修的 bug，JS 版務必照此邏輯，別用「權重=放幾份」的舊法)。
- 變數池(占位符)：`{eng}{sales}{rival}{case}{ctype}{amount}{dmg}{stat}{method}`；缺的自動兜底(隨機取當前案/對手/員工名，或通用詞)。
- 46 鍵涵蓋：event.*(11事件×含/none)、news.rival_hurt/industry、action.*(榨錢/安撫/救火/搞同行三式/發獎金…)、card.*、blame.win/fail/sue、season.terminate/explode、explode.scapegoat/eat、rival.collapse/inherit、audit.clean/dodged/fined、ending.*(survive/solo/lose_cash/lose_morale/lose_audit)。

## 8. 對手仇恨 + 反撲

- 每家 `hate` 0-100，起始 10，**隱藏**。
- **你搞他 +仇恨**(×D(grudge))：檢舉/毒模組+25、挖角+20、放假消息+12。
- **家數加碼**：`hate 底線 = 10 + (4−存活家數)×18`(越少越紅眼=困獸)。
- **消退**：沒被你搞的季 −5。
- **反撲檢定(季末)**：`d100 <= hate × D(retal)` → 反撲一次(每家每季至多一次)：
  - hate<40：放假消息 → 你信譽−(4~8)
  - 40-70：檢舉你 → **你 sin +(8~16)**(主要放血，餵你稽核)
  - >70：挖你工程師 / 搶你一個冒煙的案(困獸)
- **放血不秒殺**：反撲不直接致死，玩家要自己作死(貪過頭養大稽核炸彈)才會輸。

## 9. 對手倒閉 / 接盤 / 縮圈

- 對手 hp≤0 → 倒閉：你得 `kill_windfall × D(kill_windfall)` 接收市場，士氣+。
- 隨後政府半強迫你**接盤**一個前朝毒案(政府/銀行/S&P500案，起始風險 inherit_risk，可選臨陣脫逃)。
- 四家全倒 → **獨活勝利**(solo)。

## 10. 勝負條件

- **存活勝**：撐過第 15 季。
- **獨活勝**：四家對手全倒。
- **敗**：cash<0(現金死) / morale≤0(士氣死) / 稽核重罪(稽核死)。死因會記錄，觸發對應 ending 旁白。

## 11. 迷霧設計（UI 關鍵）

玩家**只看得到**：現金、季/行動點、士氣(●○條)、員工數與職位、案子列表(名稱/案型/🟢🟡🔴燈/下包或接盤標記)、對手血條、手牌、稽核風聲(無/隱約/逼近)。
**看不到**：所有案子的 risk/sat 數字、reputation、sin、對手 hate、audit_in 精確季數、案型隱藏參數。
→ **查核撥霧(r)** 是唯一撥開迷霧的手段。UI 要維持這個資訊不對稱，別把隱藏數字漏出來。

## 12. UI / UX 規格（RWD + PWA）

### 12.1 設計方向（定案：像素立繪 + 復古風）
- **目標視覺 = 8/16-bit 復古像素風，含像素立繪**（對手公司、員工、事件情境有像素圖，不只色塊）。
- **實作策略（單一 html，不外掛美術檔）**：像素圖以 **inline SVG** 或 **CSS box-shadow 點陣 / data-URI** 內嵌，維持單檔。起步可先用 emoji + 色塊佔位，立繪逐步替換上去（見 §12.7 資產策略）。
- **復古質感**：等寬點陣字(內嵌 web-safe 或 CSS 模擬)、`image-rendering: pixelated`、有限色盤(選一組 8-16 色 palette 全站共用)、粗邊框、掃描線/CRT 可選濾鏡、視窗式對話框(仿 JRPG 命令視窗)。
- **手機優先(mobile-first)，RWD 適配桌機**：同一套版面，桌機時內容置中、加寬留白、可放大立繪，不做兩套。

### 12.3 互動模型（重點：不打字）
- 所有動作 = **可點大按鈕**(≥44px 高，間距足夠拇指點)。
- 二次選擇(話術榨錢/安撫、選案子、選對手、選手段、甩鍋選項) = **彈出式面板 / 卡片點選**，不用輸入編號、不用下拉。
- 選案子/對手 = 直接點列表上的那張卡/那條血條。
- 一頁一焦點：出現二次選擇時用 modal 蓋住，避免同時面對太多選項。

### 12.4 資訊密度
- header 預設只顯示關鍵(季/行動點/現金/士氣/稽核風聲)；對手血條、案子列表**可摺疊**，點開才看細節。
- 旁白流保留歷史(可回捲重看笑點)，但新訊息自動捲到底。

### 12.5 回饋感（讓「快遊戲」爽）
- 擲骰動畫(d20 翻動)、大成功/大失敗特效、對手血條掉血動畫、倒閉時的爆炸/煙火、士氣條變色。
- 音效可選配(PWA 可離線包)。

### 12.7 像素資產策略（維持單一 html 的前提下做立繪）
優先序，能省則省：
1. **emoji + 色塊佔位**（第一版 mockup 就用這個，先確定版面與操作流）。
2. **inline SVG 像素圖**：用 `<svg>` 畫格子(每格一個 `<rect>`)或 `shape-rendering:crispEdges`，可放進 html 字串、可換色、檔案小。對手 logo、員工頭像、事件插圖適用。
3. **CSS box-shadow 點陣術**：一個元素用多重 box-shadow 疊出像素圖(經典技法)，純 CSS、零圖檔。
4. **data-URI base64 PNG**：真的要精緻立繪時，把小尺寸 PNG 編成 base64 內嵌(仍是單檔，但會撐大 html)。
- **共用色盤**：先定一組 palette(如 16 色)寫成 CSS variables，所有像素圖/UI 共用，復古一致性靠這個。
- **立繪清單(逐步做，非 mockup 阻塞項)**：4 家對手各一 logo/吉祥物、3 種員工(業務/PM/工程師)頭像、關鍵事件插圖(刪庫/爆炸/長官視察/勒索/獨活結局)。mockup 階段全部用 emoji 佔位即可。

### 12.6 PWA 需求
- `manifest.json`(名稱/icon/直式/standalone)、Service Worker(離線快取所有資產)。
- 目標：**手機加到主畫面像 App、離線可玩、桌機開網頁也能玩**。
- 單一 .html 起步亦可(含內嵌 CSS/JS)，PWA 化再拆 manifest+sw。

## 13. 架構（定案：純前端 A 方案，單一 html + PWA）

- **單一 `index.html`**：內嵌 CSS(`<style>`)、JS(`<script>`)、270 條旁白、像素資產。無後端、無外部相依、離線可玩。
- **檔內邏輯分區**（同一檔，用註解區塊分段，之後要拆檔也容易）：
  - `CONFIG / DIFF`：平衡數值，**對齊 Python 版**（改動時同步）。
  - `NARRATION`：270 條模板 + 加權/避近重複抽句(照 §7)。
  - `engine`：純邏輯(狀態機、擲骰、季末結算)，**不碰 DOM**——這樣能單獨跑 Monte Carlo 自測，跟 Python 版對拍驗證。
  - `ui`：DOM 渲染 + 事件綁定；每次動作後 re-render 讀 `game` 物件。
  - `pwa`：manifest(可用 data-URI 內嵌或極小外檔) + service worker 註冊。
- 狀態集中在單一 `game` 物件(對應 Python `Game`)，**單向資料流**：動作 → 改 game → re-render。
- **PWA 單檔實作**：manifest 可用 `<link rel="manifest" href="data:...">` 內嵌；service worker 需獨立檔(瀏覽器限制)——若堅持「就一個檔」，可退為「非離線的 RWD web app」，或接受 sw.js 一個小外檔。**決策點見 §16.4**。
- **無 localStorage 依賴**(單局，狀態在記憶體)；PWA 存檔續玩之後再議(要存就用 localStorage 序列化 game 物件)。
